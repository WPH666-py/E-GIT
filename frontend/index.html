📖 使用教程

基础概念

仓库（Repository）
代码的存储空间，包含项目的所有文件、文件夹和修改历史。每个仓库对应一个独立的项目。
主支（Main Branch）
仓库的默认分支，通常是项目的主线版本，如 main 或 master。
分支（Branch）
从主支分叉出来的独立开发线。可以在分支上修改代码而不影响主线，完成后可合并回去。
节点（Commit）
每次提交代码产生的记录，包含修改内容、作者、时间等信息。多个节点串联形成提交历史。
提交（Commit）
将修改保存到仓库的操作。一次提交可以包含多个文件的变更，是版本控制的核心操作。
克隆（Clone）
将远程仓库完整复制到本地的操作，获取所有文件和历史记录。
回滚（Rollback）
将分支指针退回到之前的某个提交节点，撤销后续的所有提交。回滚不会删除提交记录，只是让分支指向更早的状态。
还原（Restore）
撤销回滚操作，将分支恢复到回滚前的状态。回滚+还原可以来回切换版本。

如何绑定 GitHub 账号

  1. 打开 GitHub 官网，注册或登录你的账号。
  2. 点击右上角头像 → Settings → 左侧菜单 Developer settings → Personal access tokens → Tokens (classic)。
  3. 点击 Generate new token (classic)。
  4. 在 Note 中输入描述（如 "e-git"），设置过期时间。
  5. 勾选以下权限：
    • repo（仓库完全控制权）
    • user → read:user（读取用户信息）
    • delete_repo（删除仓库权限，如需删除功能）
  6. 点击页面底部的 Generate token，复制生成的 Token（只显示一次，请妥善保存）。
  7. 回到 e-git，点击右上角 绑定账号，输入 GitHub 用户名、邮箱和刚复制的 Token，确认绑定。
  8. 绑定成功后，右上角会显示"已绑定：你的用户名"，即可开始使用所有功能。
